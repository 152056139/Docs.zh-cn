### <a name="url-matching"></a><span data-ttu-id="e3c7d-124">URL 匹配</span><span class="sxs-lookup"><span data-stu-id="e3c7d-124">URL matching</span></span>

<span data-ttu-id="e3c7d-125">URL 匹配是一个过程，通过该过程，路由可向处理程序调度传入请求。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-125">URL matching is the process by which routing dispatches an incoming request to a *handler*.</span></span> <span data-ttu-id="e3c7d-126">此过程通常基于 URL 路径中的数据，但可进行扩展以考虑请求中的任何数据。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-126">This process is generally based on data in the URL path, but can be extended to consider any data in the request.</span></span> <span data-ttu-id="e3c7d-127">向单独的处理程序调度请求的功能是缩放应用程序的大小和复杂性的关键。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-127">The ability to dispatch requests to separate handlers is key to scaling the size and complexity of an application.</span></span>

<span data-ttu-id="e3c7d-128">传入请求将进入 `RouterMiddleware`，后者将对序列中的每个路由调用 `RouteAsync` 方法。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-128">Incoming requests enter the `RouterMiddleware`, which calls the `RouteAsync` method on each route in sequence.</span></span> <span data-ttu-id="e3c7d-129">`IRouter` 实例将选择是否通过将 `RouteContext.Handler` 设置为非空 `RequestDelegate` 来处理请求。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-129">The `IRouter` instance chooses whether to *handle* the request by setting the `RouteContext.Handler` to a non-null `RequestDelegate`.</span></span> <span data-ttu-id="e3c7d-130">如果路由为请求设置处理程序，路由处理将停止，处理程序将被调用以处理该请求。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-130">If a route sets a handler for the request, route processing stops and the handler will be invoked to process the request.</span></span> <span data-ttu-id="e3c7d-131">如果尝试了所有路由，且请求未找到任何处理程序，中间件将调用 next，请求管道中的下一个中间件将被调用。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-131">If all routes are tried and no handler is found for the request, the middleware calls *next* and the next middleware in the request pipeline is invoked.</span></span>

<span data-ttu-id="e3c7d-132">`RouteAsync` 的主要输入是与当前请求关联的 `RouteContext.HttpContext`。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-132">The primary input to `RouteAsync` is the `RouteContext.HttpContext` associated with the current request.</span></span> <span data-ttu-id="e3c7d-133">`RouteContext.Handler` 和 `RouteContext.RouteData` 是将在路由匹配后设置的输出。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-133">The `RouteContext.Handler` and `RouteContext.RouteData` are outputs that will be set after a route matches.</span></span>

<span data-ttu-id="e3c7d-134">`RouteAsync` 期间的匹配还可根据迄今完成的请求处理将 `RouteContext.RouteData` 的属性设为适当的值。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-134">A match during `RouteAsync` will also set the properties of the `RouteContext.RouteData` to appropriate values based on the request processing done so far.</span></span> <span data-ttu-id="e3c7d-135">如果路由与请求匹配，`RouteContext.RouteData` 将包含有关结果的重要状态信息。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-135">If a route matches a request, the `RouteContext.RouteData` will contain important state information about the *result*.</span></span>

<span data-ttu-id="e3c7d-136">`RouteData.Values` 是从路由中生成的路由值的字典。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-136">`RouteData.Values` is a dictionary of *route values* produced from the route.</span></span> <span data-ttu-id="e3c7d-137">这些值通常通过标记 URL 来确定，可用来接受用户输入，或者在应用程序内作出进一步的调度决策。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-137">These values are usually determined by tokenizing the URL, and can be used to accept user input, or to make further dispatching decisions inside the application.</span></span>

<span data-ttu-id="e3c7d-138">`RouteData.DataTokens` 是一个与匹配的路由相关的其他数据的属性包。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-138">`RouteData.DataTokens`  is a property bag of additional data related to the matched route.</span></span> <span data-ttu-id="e3c7d-139">提供 `DataTokens` 以支持将状态数据与每个路由相关联，以便应用程序稍后可根据所匹配的路由作出决策。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-139">`DataTokens` are provided to support associating state data with each route so the application can make decisions later based on which route matched.</span></span> <span data-ttu-id="e3c7d-140">这些值是开发者定义的，不会影响通过任何方式路由的行为。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-140">These values are developer-defined and do **not** affect the behavior of routing in any way.</span></span> <span data-ttu-id="e3c7d-141">此外，存储于数据令牌中的值可以属于任何类型，与路由值相反，后者必须能够轻松转换为字符串，或从字符串进行转换。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-141">Additionally, values stashed in data tokens can be of any type, in contrast to route values, which must be easily convertible to and from strings.</span></span>

<span data-ttu-id="e3c7d-142">`RouteData.Routers` 是参与成功匹配请求的路由的列表。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-142">`RouteData.Routers` is a list of the routes that took part in successfully matching the request.</span></span> <span data-ttu-id="e3c7d-143">路由可以嵌套在另一路由内，并且 `Routers` 属性可以通过导致匹配的逻辑路由树反映该路径。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-143">Routes can be nested inside one another, and the `Routers` property reflects the path through the logical tree of routes that resulted in a match.</span></span> <span data-ttu-id="e3c7d-144">通常情况下，`Routers` 中的第一项是路由集合，应该用于生成 URL。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-144">Generally the first item in `Routers` is the route collection, and should be used for URL generation.</span></span> <span data-ttu-id="e3c7d-145">`Routers` 中的最后一项是匹配的路由处理程序。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-145">The last item in `Routers` is the route handler that matched.</span></span>

### <a name="url-generation"></a><span data-ttu-id="e3c7d-146">URL 生成</span><span class="sxs-lookup"><span data-stu-id="e3c7d-146">URL generation</span></span>

<span data-ttu-id="e3c7d-147">URL 生成是通过其可根据一组路由值创建 URL 路径的过程。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-147">URL generation is the process by which routing can create a URL path based on a set of route values.</span></span> <span data-ttu-id="e3c7d-148">这需要考虑处理程序与访问它们的 URL 之间的逻辑分隔。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-148">This allows for a logical separation between your handlers and the URLs that access them.</span></span>

<span data-ttu-id="e3c7d-149">URL 遵循类似的迭代过程，但开头是将用户或框架代码调用到路由集合的 `GetVirtualPath` 方法中。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-149">URL generation follows a similar iterative process, but starts with user or framework code calling into the `GetVirtualPath` method of the route collection.</span></span> <span data-ttu-id="e3c7d-150">然后，每个路由会有序地调用其 `GetVirtualPath` 方法，直到返回非空的 `VirtualPathData`。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-150">Each *route* will then have its `GetVirtualPath` method called in sequence until a non-null `VirtualPathData` is returned.</span></span>

<span data-ttu-id="e3c7d-152">路由主要使用 `Values` 和 `AmbientValues` 提供的路由值来决定可能生成 URL 的位置以及要包括的值。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-152">Routes primarily use the route values provided by the `Values` and `AmbientValues` to decide where it's possible to generate a URL and what values to include.</span></span> <span data-ttu-id="e3c7d-153">`AmbientValues` 是路由值的集合，这些值是通过将当前请求与路由系统相匹配而产生的。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-153">The `AmbientValues` are the set of route values that were produced from matching the current request with the routing system.</span></span> <span data-ttu-id="e3c7d-154">与此相反，`Values` 是指定如何为当前操作生成所需 URL 的路由值。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-154">In contrast, `Values` are the route values that specify how to generate the desired URL for the current operation.</span></span> <span data-ttu-id="e3c7d-155">当路由需要获取与当前上下文关联的服务或其他数据时，需提供 `HttpContext`。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-155">The `HttpContext` is provided in case a route needs to get services or additional data associated with the current context.</span></span>

<span data-ttu-id="e3c7d-156">提示：将 `Values` 作为 `AmbientValues` 的一组替代值。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-156">Tip: Think of `Values` as being a set of overrides for the `AmbientValues`.</span></span> <span data-ttu-id="e3c7d-157">URL 生成尝试重复使用当前请求中的路由值，以便轻松使用相同路由或路由值生成链接的 URL。</span><span class="sxs-lookup"><span data-stu-id="e3c7d-157">URL generation tries to reuse route values from the current request to make it easy to generate URLs for links using the same route or route values.</span></span>
