## <a name="session-state"></a><span data-ttu-id="cb1ce-108">会话状态</span><span class="sxs-lookup"><span data-stu-id="cb1ce-108">Session state</span></span>

<span data-ttu-id="cb1ce-109">会话状态是 ASP.NET Core 中的一项功能，可用于在用户浏览 Web 应用时保存和存储用户数据。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-109">Session state is a feature in ASP.NET Core that you can use to save and store user data while the user browses your web app.</span></span> <span data-ttu-id="cb1ce-110">会话状态由服务器上的字典或哈希表组成，在浏览器的请求中保持数据。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-110">Consisting of a dictionary or hash table on the server, session state persists data across requests from a browser.</span></span> <span data-ttu-id="cb1ce-111">会话数据由缓存支持。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-111">The session data is backed by a cache.</span></span>

<span data-ttu-id="cb1ce-112">ASP.NET Core 通过向客户端提供包含会话 ID 的 Cookie 来维护会话状态，该会话 ID 与每个请求一起发送到服务器。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-112">ASP.NET Core maintains session state by giving the client a cookie that contains the session ID, which is sent to the server with each request.</span></span> <span data-ttu-id="cb1ce-113">服务器使用会话 ID 来获取会话数据。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-113">The server uses the session ID to fetch the session data.</span></span> <span data-ttu-id="cb1ce-114">因为会话 Cookie 是特定于浏览器的，所以不能跨浏览器中共享会话。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-114">Because the session cookie is specific to the browser, you cannot share sessions across browsers.</span></span> <span data-ttu-id="cb1ce-115">仅当浏览器会话结束时才能删除会话 Cookie。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-115">Session cookies are deleted only when the browser session ends.</span></span> <span data-ttu-id="cb1ce-116">如果收到过期的会话 Cookie，则创建使用相同会话 Cookie 的新会话。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-116">If a cookie is received for an expired session, a new session that uses the same session cookie is created.</span></span>

<span data-ttu-id="cb1ce-117">服务器在上次请求后保留会话的时间有限。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-117">The server retains a session for a limited time after the last request.</span></span> <span data-ttu-id="cb1ce-118">设置会话超时，或者使用 20 分钟的默认值。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-118">Either set the session timeout or use the default value of 20 minutes.</span></span> <span data-ttu-id="cb1ce-119">会话状态非常适合用于存储特定于特定会话但并不需要永久保留的用户数据。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-119">Session state is ideal for storing user data that's specific to a particular session but doesn't need to be persisted permanently.</span></span> <span data-ttu-id="cb1ce-120">在调用 `Session.Clear` 时或数据存储中会话到期时将删除后备存储中的数据。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-120">Data is deleted from the backing store either when calling `Session.Clear` or when the session expires in the data store.</span></span> <span data-ttu-id="cb1ce-121">服务器不知道关闭浏览器或删除会话 Cookie 的时间。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-121">The server doesn't know when the browser is closed or when the session cookie is deleted.</span></span>

> [!WARNING]
> <span data-ttu-id="cb1ce-122">请勿将敏感数据存储在会话中。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-122">Don't store sensitive data in session.</span></span> <span data-ttu-id="cb1ce-123">客户端可能不会关闭浏览器并清除会话 Cookie（某些浏览器在多个窗口中保持会话 Cookie）。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-123">The client might not close the browser and clear the session cookie (and some browsers keep session cookies alive across windows).</span></span> <span data-ttu-id="cb1ce-124">另外，会话可能不限于单个用户；下一个用户可能继续同一个会话。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-124">Also, a session might not be restricted to a single user; the next user might continue with the same session.</span></span>

<span data-ttu-id="cb1ce-125">内存中会话提供程序将会话数据存储在本地服务器上。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-125">The in-memory session provider stores session data on the local server.</span></span> <span data-ttu-id="cb1ce-126">如果计划在服务器场上运行 Web 应用，则必须使用粘性会话将每个会话绑定到特定的服务器上。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-126">If you plan to run your web app on a server farm, you must use sticky sessions to tie each session to a specific server.</span></span> <span data-ttu-id="cb1ce-127">Microsoft Azure 网站平台默认设置为粘性会话（应用程序请求路由或 ARR）。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-127">The Windows Azure Web Sites platform defaults to sticky sessions (Application Request Routing or ARR).</span></span> <span data-ttu-id="cb1ce-128">然而，粘性会话可能会影响可伸缩性，并使 Web 应用更新变得复杂。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-128">However, sticky sessions can affect scalability and complicate web app updates.</span></span> <span data-ttu-id="cb1ce-129">更好的选择是使用 Redis 或 SQL Server 分布式缓存，它们不需要粘性会话。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-129">A better option is to use the Redis or SQL Server distributed caches, which don't require sticky sessions.</span></span> <span data-ttu-id="cb1ce-130">有关详细信息，请参阅[使用分布式缓存](xref:performance/caching/distributed)。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-130">For more information, see [Work with a Distributed Cache](xref:performance/caching/distributed).</span></span> <span data-ttu-id="cb1ce-131">有关服务提供程序设置的详细信息，请参阅本文后续部分中的[配置会话](#configuring-session)。</span><span class="sxs-lookup"><span data-stu-id="cb1ce-131">For details on setting up service providers, see [Configuring Session](#configuring-session) later in this article.</span></span>
