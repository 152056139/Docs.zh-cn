## <a name="what-is-a-distributed-cache"></a><span data-ttu-id="1b0e6-108">什么是分布式的缓存</span><span class="sxs-lookup"><span data-stu-id="1b0e6-108">What is a distributed cache</span></span>

<span data-ttu-id="1b0e6-109">分布式的缓存共享由多个应用程序服务器 (请参阅[缓存基础知识](memory.md#caching-basics))。</span><span class="sxs-lookup"><span data-stu-id="1b0e6-109">A distributed cache is shared by multiple app servers (see [Cache Basics](memory.md#caching-basics)).</span></span> <span data-ttu-id="1b0e6-110">缓存中的信息不存储在单独的 Web 服务器的内存中，并且缓存的数据可用于所有应用服务器。这具有几个优点：</span><span class="sxs-lookup"><span data-stu-id="1b0e6-110">The information in the cache isn't stored in the memory of individual web servers, and the cached data is available to all of the app's servers.</span></span> <span data-ttu-id="1b0e6-111">这提供了几个优点：</span><span class="sxs-lookup"><span data-stu-id="1b0e6-111">This provides several advantages:</span></span>

1. <span data-ttu-id="1b0e6-112">所有 Web 服务器上的缓存数据都是一致的。</span><span class="sxs-lookup"><span data-stu-id="1b0e6-112">Cached data is coherent on all web servers.</span></span> <span data-ttu-id="1b0e6-113">用户不会因处理其请求的 Web 服务器的不同而看到不同的结果</span><span class="sxs-lookup"><span data-stu-id="1b0e6-113">Users don't see different results depending on which web server handles their request</span></span>

2. <span data-ttu-id="1b0e6-114">缓存的数据在 Web 服务器重新启动后和部署后仍然存在。</span><span class="sxs-lookup"><span data-stu-id="1b0e6-114">Cached data survives web server restarts and deployments.</span></span> <span data-ttu-id="1b0e6-115">删除或添加单独的 Web 服务器不会影响缓存。</span><span class="sxs-lookup"><span data-stu-id="1b0e6-115">Individual web servers can be removed or added without impacting the cache.</span></span>

3. <span data-ttu-id="1b0e6-116">源数据存储区具有对它 （不是使用多个内存中缓存或否缓存根本） 发出的少数几个请求。</span><span class="sxs-lookup"><span data-stu-id="1b0e6-116">The source data store has fewer requests made to it (than with multiple in-memory caches or no cache at all).</span></span>

> [!NOTE]
> <span data-ttu-id="1b0e6-117">如果使用 SQL Server 分布式缓存，则其中一些优势只有在为缓存而不是应用的源数据使用单独的数据库实例的情况下才会体现出来。</span><span class="sxs-lookup"><span data-stu-id="1b0e6-117">If using a SQL Server Distributed Cache, some of these advantages are only true if a separate database instance is used for the cache than for the app's source data.</span></span>

<span data-ttu-id="1b0e6-118">像任何缓存一样，分布式缓存可以显著提高应用的响应速度，因为通常情况下，数据从缓存中检索比从关系数据库（或 Web 服务）中检索快得多。</span><span class="sxs-lookup"><span data-stu-id="1b0e6-118">Like any cache, a distributed cache can dramatically improve an app's responsiveness, since typically data can be retrieved from the cache much faster than from a relational database (or web service).</span></span>

<span data-ttu-id="1b0e6-119">缓存配置是特定于实现的。</span><span class="sxs-lookup"><span data-stu-id="1b0e6-119">Cache configuration is implementation specific.</span></span> <span data-ttu-id="1b0e6-120">本文介绍如何配置 Redis 和 SQL Server 分布式缓存。</span><span class="sxs-lookup"><span data-stu-id="1b0e6-120">This article describes how to configure both Redis and SQL Server distributed caches.</span></span> <span data-ttu-id="1b0e6-121">无论选择哪一种实现，应用都使用通用的 `IDistributedCache` 接口与缓存交互。</span><span class="sxs-lookup"><span data-stu-id="1b0e6-121">Regardless of which implementation is selected, the app interacts with the cache using a common `IDistributedCache` interface.</span></span>
