## <a name="problem-statement"></a><span data-ttu-id="c4be2-109">问题陈述</span><span class="sxs-lookup"><span data-stu-id="c4be2-109">Problem statement</span></span>

<span data-ttu-id="c4be2-117">由于所保持的状态由服务器信任的我们预计这种状态，可能包含特定于操作环境的信息。</span><span class="sxs-lookup"><span data-stu-id="c4be2-117">Since the persisted state is trusted by the server, we anticipate that this state might contain information that is specific to the operating environment.</span></span> <span data-ttu-id="c4be2-118">这可能是或的服务器特定数据的其他部分的文件路径、 权限、 句柄或其他间接引用的形式。</span><span class="sxs-lookup"><span data-stu-id="c4be2-118">This could be in the form of a file path, a permission, a handle or other indirect reference, or some other piece of server-specific data.</span></span> <span data-ttu-id="c4be2-119">此类信息应通常不得泄露给不受信任的客户端。</span><span class="sxs-lookup"><span data-stu-id="c4be2-119">Such information should generally not be disclosed to an untrusted client.</span></span> <span data-ttu-id="c4be2-120">因此第二个要求： 保密性。</span><span class="sxs-lookup"><span data-stu-id="c4be2-120">Thus the second requirement: confidentiality.</span></span>

<span data-ttu-id="c4be2-121">最后，由于现代应用程序已被组件化，我们已了解是各个组件将想要利用此系统而不考虑其他组件在系统中。</span><span class="sxs-lookup"><span data-stu-id="c4be2-121">Finally, since modern applications are componentized, what we've seen is that individual components will want to take advantage of this system without regard to other components in the system.</span></span> <span data-ttu-id="c4be2-122">例如，如果持有者令牌组件使用此堆栈，它应运行，而不从一种反 CSRF 机制，也可能使用相同的堆栈的干扰。</span><span class="sxs-lookup"><span data-stu-id="c4be2-122">For instance, if a bearer token component is using this stack, it should operate without interference from an anti-CSRF mechanism that might also be using the same stack.</span></span> <span data-ttu-id="c4be2-123">因此最后一项要求： 隔离。</span><span class="sxs-lookup"><span data-stu-id="c4be2-123">Thus the final requirement: isolation.</span></span>

<span data-ttu-id="c4be2-124">我们可以提供进一步的约束为了缩小我们的要求的范围。</span><span class="sxs-lookup"><span data-stu-id="c4be2-124">We can provide further constraints in order to narrow the scope of our requirements.</span></span> <span data-ttu-id="c4be2-125">我们假定加密系统内运行的所有服务都都同样受信任而，数据不需要使用外部下我们直接控制服务或生成。</span><span class="sxs-lookup"><span data-stu-id="c4be2-125">We assume that all services operating within the cryptosystem are equally trusted and that the data does not need to be generated or consumed outside of the services under our direct control.</span></span> <span data-ttu-id="c4be2-126">此外，我们需要操作是尽可能快，因为 web 服务的每个请求可能会经过加密系统一个或多个时间。</span><span class="sxs-lookup"><span data-stu-id="c4be2-126">Furthermore, we require that operations are as fast as possible since each request to the web service might go through the cryptosystem one or more times.</span></span> <span data-ttu-id="c4be2-127">这样，可以对称加密适合我们的方案，而且我们可以如需要的时间之前折扣非对称加密。</span><span class="sxs-lookup"><span data-stu-id="c4be2-127">This makes symmetric cryptography ideal for our scenario, and we can discount asymmetric cryptography until such a time that it is needed.</span></span>

## <a name="design-philosophy"></a><span data-ttu-id="c4be2-128">设计理念</span><span class="sxs-lookup"><span data-stu-id="c4be2-128">Design philosophy</span></span>
