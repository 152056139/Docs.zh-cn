<span data-ttu-id="73feb-119">提示： 按照[不重复自己原则](http://deviq.com/don-t-repeat-yourself/)在您的视图。</span><span class="sxs-lookup"><span data-stu-id="73feb-119">Tip: Follow the [Don't Repeat Yourself Principle](http://deviq.com/don-t-repeat-yourself/) in your views.</span></span>

### <a name="partial-view-discovery"></a><span data-ttu-id="73feb-138">分部视图发现</span><span class="sxs-lookup"><span data-stu-id="73feb-138">Partial View Discovery</span></span>

<span data-ttu-id="73feb-139">当引用了分部视图，您可以参考以下几种方式其位置：</span><span class="sxs-lookup"><span data-stu-id="73feb-139">When referencing a partial view, you can refer to its location in several ways:</span></span>

```text
// Uses a view in current folder with this name
// If none is found, searches the Shared folder
@Html.Partial("ViewName")

// A view with this name must be in the same folder

```

<span data-ttu-id="73feb-140">你可以具有相同名称的不同分部视图不同视图文件夹中。</span><span class="sxs-lookup"><span data-stu-id="73feb-140">You can have different partial views with the same name in different view folders.</span></span> <span data-ttu-id="73feb-141">当按名称 （不带文件扩展名） 引用视图，每个文件夹中的视图将与它们位于同一文件夹中使用分部视图。</span><span class="sxs-lookup"><span data-stu-id="73feb-141">When referencing the views by name (without file extension), views in each folder will use the partial view in the same folder with them.</span></span> <span data-ttu-id="73feb-142">你还可以指定默认的分部视图，若要使用，将其放在*共享*文件夹。</span><span class="sxs-lookup"><span data-stu-id="73feb-142">You can also specify a default partial view to use, placing it in the *Shared* folder.</span></span> <span data-ttu-id="73feb-143">共享的分部视图将使用由任何没有其自己版本的分部视图的视图。</span><span class="sxs-lookup"><span data-stu-id="73feb-143">The shared partial view will be used by any views that don't have their own version of the partial view.</span></span> <span data-ttu-id="73feb-144">你可以有一个默认分部视图 (在*共享*)，该方法由具有与父视图相同的文件夹中的相同名称的部分视图重写。</span><span class="sxs-lookup"><span data-stu-id="73feb-144">You can have a default partial view (in *Shared*), which is overridden by a partial view with the same name in the same folder as the parent view.</span></span>
